Apache Airflow overview

  • open source workflow orchestration tool
  • build and run workflows
  • workflow represented as DAG (Directed Acyclic Graphs)
  • AirFlow not a streaming solution

Architecture

DAG illustration

  • specifies dependancy within tasks
  • in order to execute them

Airflow task state

  • scheduled
  • removed
  • queued
  • running
  • success
  • failed
  • up for retry

Features

  • Pure [[python]]
  • useful UI (get insights)
  • Integration (plug and play)
  • Easy to use (pipeline scope)
  • Open source

Principles

  • Scalable
  • Dynamic
  • Extensible
  • Lean (explicit)

DAG

  • Directed Acyclic Graph (each edge with direction)
  • Acyclic - no loops (cycles)
  • Nodes are tasks
  • Edges are dependencies
  • Defined as code in python also scheduled

Tasks and Operator

  • Tasks are written in python
  • Tasks implement operator, ex., python, sql and bash operators
  • Operators determine what each task does
  • Sensor operators poll for certain time or condition
  • Other operators include email and http request operators

DAG definition components

  1. Library imports
  2. DAG arguments
  3. DAG definition
  4. Task definitions
  5. Task pipeline

Airflow Scheduler

  • Deploys on worker array
  • follows your DAG
  • first DAG run
  • Subsequent runs

Advantage of workflows as code

  • Maintainable
  • Versionable
  • Collaborative
  • Testable

DAGs View

  • Rich UI that simplifies working with data pipelines
